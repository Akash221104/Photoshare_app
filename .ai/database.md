# Database Schema and Repositories

The system runs on a serverless PostgreSQL database (on Neon) and utilizes a custom repository layer with raw parameterized queries executed via a singleton connection pool.

---

## Database Table Catalog

### 1. `users`
Stores user profile information.
- `id` (UUID, Primary Key): Autogenerated default `uuid_generate_v4()`.
- `name` (VARCHAR): Display name.
- `email` (VARCHAR, Unique): User email address.
- `email_verified` (BOOLEAN): Defaults to `FALSE`.
- `image` (TEXT): Profile photo URL.
- `created_at` / `updated_at` (TIMESTAMP WITH TIME ZONE).
- *Index:* `idx_users_email` (Unique index on `email`).

### 2. `sessions`
Used by Neon Auth / Better Auth to manage user session tokens.
- `id` (VARCHAR, Primary Key): Session ID.
- `token` (VARCHAR, Unique): Opaque token.
- `user_id` (UUID): Foreign Key referencing `users(id)` with cascading deletion.
- `expires_at` (TIMESTAMP WITH TIME ZONE).
- `ip_address` (VARCHAR).
- `user_agent` (TEXT).
- *Index:* `idx_sessions_user_id`, `idx_sessions_token`.

### 3. `accounts`
Handles third-party social provider link accounts (e.g., Google OAuth).
- `id` (VARCHAR, Primary Key).
- `account_id` / `provider_id` (VARCHAR).
- `user_id` (UUID): Foreign Key referencing `users(id)`.
- `password` (TEXT): Hashed credentials.
- `access_token` / `refresh_token` / `id_token` (TEXT).
- `access_token_expires_at` / `refresh_token_expires_at` (TIMESTAMP).
- *Unique Constraint:* `(provider_id, account_id)`.

### 4. `events`
Stores information about created galleries.
- `id` (UUID, Primary Key).
- `name` (VARCHAR).
- `description` (TEXT).
- `join_code` (VARCHAR, Unique): Random alphanumeric string.
- `host_id` (UUID): Foreign Key referencing `users(id)`.
- `upload_mode` (VARCHAR): Constraint check `IN ('ALL', 'HOST_ONLY')`. Defaults to `'ALL'`.

### 5. `event_members`
A join table representing many-to-many relationship of users in events.
- `id` (UUID, Primary Key).
- `event_id` (UUID): FK referencing `events(id)`.
- `user_id` (UUID): FK referencing `users(id)`.
- `role` (VARCHAR): Constraint check `IN ('host', 'guest', 'admin')`. Defaults to `'guest'`.
- *Unique Constraint:* `(event_id, user_id)`.

### 6. `photos`
Event photo metadata.
- `id` (UUID, Primary Key).
- `event_id` (UUID): FK referencing `events(id)`.
- `uploaded_by` (UUID): FK referencing `users(id)`.
- `cloudinary_public_id` (VARCHAR) / `cloudinary_url` (TEXT).
- `width` / `height` (INT).
- `status` (VARCHAR): Old legacy column `chk_photo_status` (`processing`, `processed`, `failed`).
- `processing_status` (VARCHAR): Pipeline tracking status `chk_processing_status` (`PENDING`, `PROCESSING`, `COMPLETED`, `FAILED`).
- `processing_started_at` / `processing_completed_at` (TIMESTAMP).
- `processing_error` (TEXT).

### 7. `photo_faces`
Detailed faces detected in uploaded event photos.
- `id` (UUID, Primary Key).
- `photo_id` (UUID): FK referencing `photos(id)`.
- `box_x1` / `box_y1` / `box_x2` / `box_y2` (INT): Scaled bounding box coordinates.
- `x` / `y` / `width` / `height` (INT): Base bounding box details.
- `confidence` (DOUBLE PRECISION): Facial detection confidence.
- `embedding` (VECTOR(512) or double precision[]): Facial vector representation.
- `yaw` / `pitch` / `roll` / `blur` / `brightness` / `sharpness` / `face_quality` / `occlusion_score` (DOUBLE PRECISION): InsightFace quality and pose vectors.
- `crop_url` (TEXT): Separate Cloudinary face crop if saved.

### 8. `selfies`
User facial embeddings representing user matching profile for an event.
- `id` (UUID, Primary Key).
- `user_id` (UUID): FK referencing `users(id)`.
- `event_id` (UUID): FK referencing `events(id)`.
- `cloudinary_public_id` (VARCHAR) / `cloudinary_url` (TEXT).
- `embedding` (VECTOR(512) or double precision[]).
- *Unique Constraint:* `(user_id, event_id)`.

### 9. `user_feedback`
Stores feedback matching validation.
- `id` (UUID, Primary Key).
- `user_id` / `photo_id` / `face_id` (FKs).
- `feedback_type` (VARCHAR): Constraint check `IN ('is_me', 'not_me')`.

## Repositories Layer
1. **`PhotoRepository`** (`database/repositories/photo.repository.ts`): Creates, reads, updates, and deletes event photos.
2. **`EventRepository`** (`database/repositories/event.repository.ts`): Queries event details and membership list.
3. **`EmbeddingRepository`** (`repositories/embedding.repository.ts`): Handles `photo_faces` and pipeline locks.
4. **`SearchRepository`** (`repositories/search.repository.ts`): Implements pgvector native similarity match query and Javascript array-based fallback calculations.
5. **`SelfieRepository`** (`repositories/selfie.repository.ts`): Manages selfie records.
6. **`DownloadRepository`** (`repositories/download.repository.ts`): Handles queries for fetching batch images to export as ZIP folders.
