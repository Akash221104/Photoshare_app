# AI Memory System

This folder contains the permanent AI memory system for this repository. It is designed to help future AI assistants quickly understand the architecture, tech stack, APIs, database, and features of the codebase without needing to re-analyze it from scratch.

## File Map

- `project.md`: Overall project goals and overview.
- `architecture.md`: High-level system architecture and data flows.
- `tech_stack.md`: Frontend, backend, database, and infrastructure technologies.
- `folder_structure.md`: Key directories and files layout.
- `api.md`: REST API design and endpoint catalog.
- `database.md`: Schema, tables, relations, and transactional practices.
- `authentication.md`: Auth architecture (Neon Auth, proxy, client-side session).
- `ai_pipeline.md`: AI workflows, face recognition, search, and embeddings.
- `routing.md`: Next.js App Router structure and page hierarchy.
- `components.md`: UI components catalog and design guidelines.
- `state_management.md`: Client and server state management (TanStack Query, hooks).
- `configuration.md`: Configuration files and compilation setups.
- `environment.md`: Environment variables description.
- `dependencies.md`: NPM and external library details.
- `current_status.md`: Development state and active sprint tasks.
- `completed_features.md`: Implemented and verified business features.
- `pending_features.md`: Future roadmaps and backlogs.
- `known_issues.md`: Bugs, limits, and gotchas.
- `decisions.md`: Architecture Decisions Log (ADRs).
- `conventions.md`: Code style, linting, and design patterns.
- `changelog.md`: History of major repository modifications.
- `glossary.md`: Core domain vocabulary and definitions.

## Memory Modules

Detailed domain memories are kept in the `/memory/` subdirectories:
- `modules/`: Core backend and domain logic modules.
- `pages/`: Page-by-page rendering and interactivity details.
- `services/`: API and third-party business services.
- `api/`: Endpoint implementations.
- `database/`: Repository layers and SQL patterns.
- `ui/`: Design system token mappings and UI primitives.
- `features/`: Multi-part functional user journeys.
